Overview

The goal of this project is to modernize and simplify the DMETrack order management dashboard to provide only information relevant to a user's role presented in an easy-to-digest manner.

These prototypes demonstrate the content that would be available and allow users to interact with a limited set of functionality for testing purposes.

New Concepts

Roles

We're updating DMETrack to better assign users to roles, which will allow us to customize what content is displayed and unlock different sets of functionality.

The prototypes here demonstrate what the grid would look and act like for users with Dispatch and Site Manager roles. The content and functionality available to these roles within the grid is the same. Differences come later as additional features — such as warehouse zoning — are introduced.

Assignment

The new Assignment field is used to indicate which user or department is responsible for an order. When the scheduling team has finalized the schedule for an order and it's ready for fulfillment, they will assign it to Dispatch. It will then show up on the dispatch board.

If an order is reset or needs to be rescheduled, the assignment is cleared so it goes back to the scheduling board.

Tagging

Tags are used to display attributes about the order, patient, or client. They can be used to filter and sort orders in the grid.

A dedicated tagging mechanism replaces the previous system of using values in the driver and route fields for categorization. This dedicated setup allows those fields to be used only for their intended purpose and also allows for adding multiple tags to an order.

Filtering

Standard Filters

Standard filters can be added or removed using the filter button in the toolbar. When filters are applied, a red dot is displayed on the filter button.

Available filters

  • Warehouses
  • Order status
  • Schedule date
  • Order reason
  • Order priority
  • Driver
  • Route
  • Tags

Behavior

  • Warehouse, driver, and tags filters are searchable autocompletes.
  • Order priority, reason, and route status are basic dropdowns.
  • All dropdown and autocomplete inputs allow selecting multiple options.
  • Order status and schedule date are filter chips where the user can select one or more options. At least one selection is required.

Order status options

  • Open: Orders in Received, Acknowledged, or En Route status with a confirmed schedule that are assigned to the Dispatch queue
  • Closed last 48 hours: Orders completed within the last 48 hours

Schedule date options

  • Yesterday
  • Today
  • Tomorrow

To prevent latency of these critical functions, filtering on order status and schedule date is limited to a smaller subset of orders. This dashboard is intended for day-to-day order fulfillment operations. A wider selection of orders can be searched using a separate order report.

Note Filtering is not enabled in this limited prototype. It simply displays an example of the available filters.

Quick Filters

Quick filters are buttons that allow the user to further narrow down the results returned from the standard filters.

Available filters

  • STAT / Urgent
  • SLA approaching
  • Missed schedule
  • Delayed
  • Rescheduled
  • New note

Behavior

  • Mutually exclusive (only one can be active at a time)
  • Clicking an active filter toggles it off
  • Grid rows are shown/hidden by predicate logic tied to each filter
  • Each filter displays a badge with the number of orders in the list that match the filter criteria

Applied Filter Chips

Behavior

  • Chips are rendered to display the active filter values
  • Individual filter values can be removed using the included remove buttons

Note Filtering is not enabled in this limited prototype. It simply displays an example of the applied filters.

Grid Actions

Edit Driver

When hovering over a Driver cell, a pencil icon is displayed. Clicking this enters the cell into edit mode.

Users can search a list of drivers and select one to assign. They may also remove the assigned driver.

Drivers are grouped by warehouse for easier selection. (A single driver may appear under multiple warehouses.)

Note How drivers and warehouses are sorted is to be determined. Some ideas include sorting the warehouse groups by distance from ordering warehouse, sorting the order warehouse to the top then listing other warehouses after, only displaying warehouses zoned for the order address area, or simply alphabetizing.

Edit Route

When hovering over a Route cell, a pencil icon is displayed. Clicking this enters the cell into edit mode.

Users will be able to select a route from a list of available routes. They may also remove the assigned route.

Note Route selection UI is still in development.

Edit Tags

When hovering over a Tags cell, a pencil icon is displayed. Clicking this enters the cell into edit mode.

Users can search a list of tags and select one or more to apply. They may also remove tags.

Note The list of tags presented here is an example, not a comprehensive or finalized list of tags that will be available.

More Actions Menu

Clicking the 3 dots icon on the far right opens the row action menu.

Available actions:

Edit driver / route
Opens a modal to allow editing of the driver, route, and tag values at one time. Preview only
Add on
Place the order on an existing route as an add on. Preview only
Reset order
Clear the order schedule and reset it to Received/Acknowledged status. Preview only
View equipment
Slides out a view to see the equipment available on the order.
  • Individual sections for delivery, pickup, and service equipment (when applicable)
  • Displays equipment item name and quantity
  • Service items display the service details selected by the client during ordering when available
  • "View order" button takes the user to the Edit Order page for more details. Preview only
