Overview

The goal of this project is to modernize and simplify the DMETrack order management dashboard to provide only information relevant to a user's role presented in an easy-to-digest manner.

These prototypes demonstrate the content that would be available and allow users to interact with a limited set of functionality for testing purposes.

New Concepts

Roles

We're updating DMETrack to better assign users to roles, which will allow us to customize what content is displayed and unlock different sets of functionality.

The prototypes here demonstrate what the grid would look and act like for users with the Scheduling role alone as well as users with both Scheduling and Manager roles.

Assignment

Scheduling users can claim orders to work on by assigning the order to themselves. This replaces the previous system of using the unreliable Viewed By data with a dedicated assignment mechanism or using the driver and route tags to indicate which department should work an order. Once claimed, the order is removed from the scheduling queue and added to the user's personal list of orders.

Users can also assign orders to other departments to move them from the scheduling queue to the appropriate department queue. Other departments in scope are Respiratory Therapy, Purchasing, and Dispatch.

The average scheduling user can only assign an order to themselves or another department. Scheduling users with manager roles have additional capabilities for assigning orders to other users or clearing the assignment to move the order back to the unassigned queue.

Tagging

Tags are used to display attributes about the order, patient, or client. They can be used to filter and sort orders in the grid.

A dedicated tagging mechanism replaces the previous system of using values in the driver and route fields for categorization. This dedicated setup allows those fields to be used only for their intended purpose and also allows for adding multiple tags to an order.

Filtering

Standard Filters

Standard filters can be added or removed using the filter button in the toolbar.

Available filters

  • Warehouses
  • Order type
  • Order priority
  • Order reason
  • Schedule status
  • Tags
  • Assigned to

Behavior

  • Warehouse and tags filters are searchable autocompletes.
  • Order type, priority, reason, and schedule status are basic dropdowns.
  • All dropdown and autocomplete inputs allow selecting multiple options.

The "Assigned to" field changes depending on the user's role:

  • Standard scheduling users can select "Me" and/or "Unassigned" to view orders they have claimed and/or orders that have not been claimed yet. At least one selection is required.
  • Scheduling manager users see a searchable dropdown where they can select one or more scheduling users whose orders they would like to view and/or unassigned orders. If left blank, all orders are shown.

Note Filtering is not enabled in this limited prototype. It simply displays an example of the available filters.

Quick Filters

Quick filters are buttons that allow the user to further narrow down the results returned from the standard filters.

Available filters

  • Rescheduled
  • STAT/Urgent
  • New note

Behavior

  • Mutually exclusive (only one can be active at a time)
  • Clicking an active filter toggles it off
  • Grid rows are shown/hidden by predicate logic tied to each filter
  • Each filter displays a badge with the number of orders in the list that match the filter criteria
  • Count badges are static examples in this prototype

Applied Filter Chips

Behavior

  • Chips are rendered to display the active filter values
  • Individual filter values can be removed using the included remove buttons

Note Filtering is not enabled in this limited prototype. It simply displays an example of the applied filters.

Grid Actions

Edit Assignee

When hovering over an Assignee cell, a pencil icon is displayed. Clicking this enters the cell into edit mode.

Standard schedulers
  • Basic dropdown with limited options
  • Can select "Me" to claim the order for working
  • Cannot assign to other users or unassign the order from themselves
Scheduling managers
  • Searchable dropdown listing all schedulers
  • Can assign the order to themselves or search a list of scheduling users to someone else
  • Can also clear the assignment to send the order back to the scheduling queue
All schedulers
  • Can assign the order to other user groups representing different departments
  • This removes the order from the scheduling queue and adds it to the boards for the selected user group

Department groups

Orders can be assigned to Respiratory Therapy, Purchasing, or Dispatch. Orders can only be assigned to Dispatch if they have a schedule set and are in the Confirmed or Rescheduled schedule status.

Edit Tags

When hovering over a Tags cell, a pencil icon is displayed. Clicking this enters the cell into edit mode. Users can search a list of tags and select one or more to apply. They may also remove tags.

Note The list of tags presented here is an example, not a comprehensive or finalized list of tags that will be available.

More Actions Menu

Clicking the 3 dots icon on the far right opens the row action menu.

Schedule order
Respond to the schedule request or set a schedule for the order. Preview only
Reset order
Clear the order schedule and reset it to Received/Acknowledged status. Preview only
View equipment
Slides out a view to see the equipment available on the order.
  • Individual sections for delivery, pickup, and service equipment (when applicable)
  • Displays equipment item name and quantity
  • Service items display the service details selected by the client during ordering when available
  • "View order" button takes the user to the Edit Order page for more details. Preview only
