Overview and Goals

The goal of this project is to modernize and simplify the DMETrack order management dashboard to provide only information relevant to a user's role presented in an easy-to-digest manner.

These prototypes demonstrate the content that would be available and allow users to interact with a limited set of functionality for testing purposes.

Roles

We're updating DMETrack to better assign users to roles, which will allow us to customize what content is displayed and unlock different sets of functionality.

The prototypes here demonstrate what the grid would look and act like for users with the Scheduling role alone as well as users with both Scheduling and Manager roles.

Filtering

Standard Filters

Scheduling users can filter on the following fields:

  • Warehouses
  • Order type
  • Order priority
  • Order reason
  • Schedule status
  • Tags
  • Assigned to

All dropdown and autocomplete inputs allow selecting multiple options.

The "Assigned to" field changes depending on the user's role:

  • Standard scheduling users can select "Me" and/or "Unassigned" to view orders they have claimed and/or orders that have not been claimed yet.
  • Scheduling manager users see a searchable dropdown where they can select the scheduling users whose they would like to view and/or unassigned orders.

Note: Filtering is not enabled in this limited prototype. It simply displays an example of the available filters.

Quick Filters

Quick filters are buttons that allow the user to further narrow down the results returned from the standard filters.

Available quick filters:

  • Rescheduled
  • STAT/Urgent
  • New note

Behavior:

  • Mutually exclusive (only one can be active at a time)
  • Clicking an active filter toggles it off
  • Grid rows are shown/hidden by predicate logic tied to each filter
  • Each filter displays a badge with the number of orders in the list that match the filter criteria

Applied Filter Chips

Behavior:

  • Chips are rendered to display the active filter values
  • Individual filter values can be removed using the included remove buttons

Note: Filtering is not enabled in this limited prototype. It simply displays an example of the applied filters.

Table Toolbar Actions

  • Download: Download a spreadsheet of the current order results (display only, not currently functional)
  • Order history: View a list of the last 5 orders changed by the user
    • Error prevention should the user move the order into the dispatch queue prematurely
    • Displays high-level order details and links to the Edit Order page and Patient Profile
  • Column customization: Allows the user to show or hide columns (display only, not currently functional)
  • Table customization: Allows the user to select a density for the table data.
    • Condensed: smaller content with less space around it
    • Standard: default content size and spacing
    • Relaxed: default content size with more space around it

Grid Columns

All Scheduling Users

Visible columns:

  • Assignee
  • Order
  • Priority / Reason
  • Patient
  • Ordered
  • Requested
  • Scheduled
  • Schedule Status
  • Tags
  • Actions

What each column shows:

  • Assignee:
    • Name of the user who is working the order (blank when unassigned)
    • Edit pencil button on hover to allow changing assignee (see interactive features below)
  • Order:
    • Order ID (links to Edit Order page)
    • Flashing "New Note" icon when applicable
    • Order type
  • Priority / Reason:
    • Order priority (STAT and Urgent emphasized in red)
    • Order reason
  • Patient:
    • Patient name (links to Patient Profile)
  • Ordered:
    • Date and time order was placed (dash if empty)
  • Requested:
    • Date and time window requested by client, patient, or caregiver (dash if empty)
  • Scheduled:
    • Date and time window of order schedule (dash if empty)
  • Schedule Status:
    • Color-coded badge showing where the order is in the scheduling process
  • Tags:
    • Badges displaying attributes about the order, patient, or client
    • Some badges are sorted to the front when present, then all remaning badges are displayed alphabetically
      1. Time-Sensitive
      2. Facility
      3. Service
      4. Liquid O2
      5. All other tags in alphabetical order
    • Edit pencil button on hover to add or remove tags
    • When more than 4 tags are displayed, they are truncated with text displaying how many more tags are present.
    • Hovering over this text displays a popup listing the other tags.
  • Actions:
    • More actions menu button (...)
    • Expand/collapse toggle button

Scheduling Manager Users

In addition to the content above, manager users have an additional column in the header and each row containing a checkbox. This will allow them to perform bulk actions.

Expanded Row Content

The chevron button on the far right of the row allows the users to expand or collapse more content about the order.

When a row is expanded, two detail sections are displayed:

  • Client:
    • Corporate client name
    • Branch name
  • Provider:
    • Warehouse name

Grid Actions

Edit Assignee

When hovering over the Assignee row, a pencil icon is displayed. Clicking this enters the cell into edit mode.

Standard scheduling users can select "Me" to claim the order for working. They cannot assign to other users or unassign the order from themselves.

Scheduling manager users can assign the order to themselves or search a list of scheduling users to someone else. They can also clear the assignment.

All scheduling users can also assign the order to other user groups so the order is added to the boards for that user type and removed from the scheduling queue.

  • Purchasing
  • Respiratory Therapy
  • Dispatch

Orders can only be assigned to Dispatch if they have a schedule set and are in the Confirmed or Rescheduled schedule status.

Edit Tags

When hovering over the Tags row, a pencil icon is displayed. Clicking this enters the cell into edit mode.

Users can search a list of tags and select one or more to apply. They may also remove tags.

Note: The list of tags presented here is an example, not a comprehensive or finalized list of tags that will be available.

More Actions Menu

Clicking the 3 dots icon on the far right opens the row action menu.

Available actions:

  • Schedule order: Respond to the schedule request or set a schedule for the order (display only, not currently functional)
  • Reset order: Clear the order schedule and reset it to Received/Acknowledged status (display only, not currently functional)
  • View equipment: Slides out a view to see the equipment available on the order.
    • Individual sections for delivery, pickup, and service equipment (when applicable)
    • Displays equipment item name and quantity
    • Service items display the service details selected by the client during ordering when available
    • "View order" button takes the user to the Edit Order page for more details (display only, not currently functional)
